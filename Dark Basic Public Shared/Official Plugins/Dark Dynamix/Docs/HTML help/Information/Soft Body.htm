Soft Bodies


What's a Soft Body?

The soft body feature of the NVIDIA PhysX SDK allows the simulation of volumetric deformable objects. It can also be used for items that are not usually thought of as classical soft bodies, such as plants or multiple layers of cloth. Here are some examples:

Tires of a monster truck
Big fat guys, monsters
Curvy girls
Objects with complex geometry such as trees, grass

To simulate complex geometry like a tree with many branches and leaves, a coarser volumetric mesh is first build around the object. The volumetric mesh is then simulated and moves the complex geometry with it. The advantage of the technique is that it makes the simulation of triangle soups possible, i.e. every possible mesh can be simulated.

The soft body feature is very similar to the cloth feature. Therefore, the two features have similar parameters and methods.

Content Pipeline

The content pipeline for soft bodies has some additional stages to the cloth pipeline.

1. The artist creates a detailed triangle mesh, possibly with textures, bump maps etc.
2. A tool is used to create a coarser tetrahedral mesh around the triangle mesh. (PhysX Viewer - You will need to download the PhysX SDK(2.8.4) which will include this tool)
3. The tetrahedral mesh is simulated using the soft body(and soft body mesh) commands.
4. The original triangle mesh is deformed along with the tetrahedral mesh by the user.
5. The triangle mesh is displayed on the screen.

The process works for arbitrary triangle soups like the palm tree. The image on the right shows how the original triangle soup is embedded in the tetrahedral mesh which is shown in wireframe mode.



The resolution of the tetrahedral mesh can be chosen independently of the resolution of the triangle mesh. This allows for choosing a level of detail for the simulation without losing visual resolution. The following images show different resolutions of a palm tree mesh. PhysX Viewer


The PhysX viewer is a very useful tool for creating the tet files needed to create a soft body mesh, the viewer gives you control over how detailed you want the soft body mesh to be, this does not affect how it looks but how detailed the simulation is for that particular model. It is important to get the detail level correct as soft bodies are very CPU intensive and it will not always be possible to use a very detailed soft body mesh.

The PhysX viewer can import obj files, if you look around the TGC forum you will find obj-x and x-obj converters which will be needed to get your x/dbo models set up as a soft body mesh.

You will need to sign up as a NVIDIA PhysX developer and download the Physx SDK(2.8.4), you will find the viewer in the 'bin' folder, there is also lots of models included which you can play around with.
